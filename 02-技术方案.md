# 02 · 技术方案

## 一、插件架构

**Anima Prompt 插件** — ComfyUI custom nodes

- **本地路径**：`D:\ai painting\ComfyUI-aki-v2\...\custom_nodes\`
- **Git 归档**：`E:\new项目\`
- **GitHub**：[kiriasuka151010-oss](https://github.com/kiriasuka151010-oss)

### 核心节点

| 节点 | 功能 |
|------|------|
| DanbooruBrowser | D站标签搜索与浏览，拉取5类标签 |
| EditablePrompt | 可编辑提示词，支持下拉选取，不手输 |
| MooshiePipeline | 标签→Prompt→生成 一体化管线 |

- 路由前缀 `/mooshie/` 避免与原版 D站插件冲突
- Widget 跨节点传播：JS 直写（ComfyUI 不自动传播 upstream 值）

## 二、工作流

```
D站标签搜索 → 标签过滤/组合 → Prompt编码 → Anima生成 → LoRA链式应用 → 输出
```

### 模型加载

非 CheckpointLoaderSimple，使用分体加载：

| 组件 | 模型 |
|------|------|
| UNET | animabaseV10 / preview3Base |
| CLIP | qwen_3_06b_base |
| VAE | qwen_image_vae |
| LoRA | 角色/风格 LoRA（链式应用） |

## 三、D站 API 要点

- User-Agent: `DanbooruAnimaPrompt/1.0`
- 原生5类标签：`tag_string_artist` / `character` / `copyright` / `general` / `meta`
- 返回格式：JSON，包含完整标签树

## 四、跨模型熟悉度

| 模型 | 用途 | 熟练度 |
|------|------|--------|
| Anima (animabaseV10) | 二次元主模型 | ★★★★ |

## 五、已解决的技术问题

- Widget 跨节点传播（JS 直写下游 widget）
- ComfyUI 路由冲突（独立前缀 `/mooshie/`）
- D站 API 五类标签映射与编码
- LoRA 链式应用与权重调参
- UI 偏好：端口能少就少，内部 widget 能藏就藏（0px）
