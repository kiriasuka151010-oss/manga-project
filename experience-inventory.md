## 二、九瓴科技 · AI 产品经理实习生
*2025.11 – 2026.01*

### 实际做了什么

#### 1. 长视频 AI 工作流搭建

**产品逻辑**：用户选题材类型 → 一句话输入 → 自动产出 1-2 分钟长视频

**竞品对标**：Motion（含历史人物、科普教育、节日庆祝、悬疑短剧等多个分类 + 多种画风 demo）

**工作流全链路**（全权负责）：
```
一句话 → 扩写故事梗概 → 扩写完整脚本 → 
  ├─ 一致性要求高 → 生资产（角色/场景）
  ├─ 文生视频 / 图生视频
  └─ 宫格生视频 / 首尾帧
```

**负责的题材**：写实历史人物（2条）、节日庆祝（1条）、数字人口播（1条），共4条 demo 视频

**协作方式**：
- 先在文档里手搓视频 demo，反复调试
- 开发团队将文档粗略翻译成 Coze 工作流
- 再在 Coze 上对提示词和流程调优

**效果**：写实历史人物两条效果不错——约 60-70 次生成才能挑出一条可用画面。上线后整体效果一般。

**最佳 demo**：伊丽莎白一世——展现人物从幼年到标志性老年形象的成长弧线，而非静态肖像。乔布斯则锁死标志性长相（黑色高领衫、圆框眼镜）。

**可复用题材归纳**（竞品 Motion 分析后总结）：

| 题材 | 适配画风 |
|------|---------|
| 历史人物 | 写实 |
| 科普向 | 写实 / 迪士尼 |
| 新闻数字人口播 | 写实 |
| 节日祝福 | 迪士尼 / 日漫 |
| 故事向短篇 | 日漫 / 美漫 |
| 悬疑恐怖 | 写实 / 美漫 |

#### 3. 宫格分镜控制技巧

- **原理**：将九宫格或四宫格图作为输入，告知 AI 按分镜图/故事板生成，每格配具体分镜提示词
- **效果**：AI 遵循度提升、镜头丰富度增加、角色一致性更好维护
- **来源**：产品经理咨询 Sora 团队 + 行业内漫剧创作者共享的实践经验
- **局限**：Seedance 2.0 已原生支持连贯切镜，不再需要此技巧
- **保留**：留有宫格测试文档

#### 4. Agent 爆款复刻（了解但未直接负责）

- 设想：输入 TikTok/YouTube 链接 → 提取笑点核心 → 复刻生成
- 目的：为 Agent 自动化服务
- 本人未直接负责，但理解全流程

Leader 觉得被威胁，未看到最终上线效果。根据后续同事反馈，上线后效果一般，与自己离开时差别不大。

### 面试可深挖的点

- 宫格分镜的工程思考——如果 Seedance 2.0 让它过时了，那这个经验的价值在哪？（答：体现了"在模型能力不足时用 Prompt 技巧弥补"的问题解决思维）

## 三、元象科技 · AI 视频游戏部 产品岗
*2026.01 – 2026.05*

### 角色与定位

- 三段实习中待最久、权位最重、沟通最深的一段
- 获得上级信任，实质参与决策
- 入场即为"救火"——接手视频平台调优的遗留问题

### 全流程时间线

#### Phase 1：救火 · 视频平台调优（2026.02，年前）

- 元象自研了视频平台，但 Sora 相关的很多具体问题未调优好
- 入场后继续帮视频平台调优，解决 Sora 模型的诸多实际问题

#### Phase 2：Seedance 2.0 颠覆（年后）

- **巨变**：不再需要宫格解决动态问题，不再出静态 PPT，提示词遵循度大幅上升
- 简单的 prompt 就能产出高质量视频
- **但好景不长**：Seedance 2.0 出现"降智"（字节侧问题），能力下降 + 未开放 API → 只能在即梦平台排队使用

#### Phase 3：游戏 CG 突破（关键转折）

- 在模型降智、排队受限的情况下，**产出极高质量的游戏画风 CG**
- 虽只是单纯 15s 片段拼接，但画面质量令大领导刮目相看
- **领导看到新模型潜力后，决定全面对接 Seedance 2.0 到自研平台**

#### Phase 4：平台全面调优（全流程参与）

调优维度（手工测试为主，文档输出，无 coding）：

| 调优项 | 具体内容 |
|--------|---------|
| 尺度边界测试 | 改写敏感词，测试能出多色情的内容边界 |
| 版权测试 | 真人审核过滤 |
| 15s 连贯性 | 片段之间的衔接流畅度 |
| 人物/场景一致性 | 站位、场景、声音的一致性控制 |
| 提示词入参结构 | 输入格式的标准化 |
| 人物设定图入参结构 | 角色参考图的输入规范 |

每个环节都有参与，通过文档修改递交给 Leader。

#### Phase 5：漫剧内容生产

- 效果达标后，开始招聘 AI 编剧等，正式进行漫剧生产
- 初期：边协助测试平台问题边产出内容，剧本和灵感自己找
- **效率瓶颈**：后发现效率太低，难以流水化
- **Saylo 管线创新**：利用公司已有的 Saylo（AI 陪伴聊天产品），将聊天内容作为素材库 → LLM 改写为剧本 → 挑选产出漫剧
- 用户手上保留一部自己参与的漫剧

#### Phase 6：漫剧商业化失败

- 漫剧直接上到 Saylo 新板块
- 平台效果一般
- **AI 视频成本太高，入不敷出，被迫放弃 AI 漫剧路线**
- 到 5 月，团队整体消极，方向明显做不下去

#### Phase 7：转图文方向（2026.05）

- 领导决定向图文靠拢：生图成本低

### 商业逻辑与公司架构经验

这些来自三段实习的观察，面试中展现商业成熟度时可用：

1. **"谁买单"问题** — 评判内容好坏的人往往不是受众。内部觉得 OK，受众觉得垃圾。负责人困惑"为什么没人看"，但从未想过评判标准本身就是错的。

2. **内容质量无人把关，冷启动无正反馈** — 团队一言堂定方向，缺乏数据驱动的反馈闭环。做了东西没人看，不知道为什么，也不知道怎么改。

3. **产品与技术混淆** — 产品无法判断 AI 技术能做到什么程度、怎么做，最后技术直接掌管内容方向。传统内容生产者无法诊断"哪里出问题、怎么解决"，被迫退位。

4. **急于流水化卖铲子，ToC/ToB 理不清** — 内容本身还不行就急着做平台。"中小非专业有创作需求者"这个客户画像可能根本不存在，即便存在也转移成本极低——谁好用转谁，谁能白嫖用谁。专业创作者有自己的工具链（如剪映），普通用户直接用抖音的+号发布。

5. **内容才是王道，擦边不持久** — 黄色擦边能解一时渴，沉淀不了长期价值。前期擦边流量好，禁黄后数据骤降。必须研究怎么生产真正的好故事、好内容。

6. **AI 视频还没到商业化临界点** — 所有人都觉得 AI 视频是机遇，但实际做的是"次级调优"——花三个月优化的东西，半年后就被模型迭代覆盖了。怎么调优也难以突破商业化，最终还是需要传统人工剪辑和修复。

7. **跨界幻觉** — 电商、AI 陪聊、数据分析等领域的公司，都觉得自己的垂直经验能在 AI 内容赛道占优。实际从成本到收入很难回本。领域优势不直接等于内容优势。

## 四、项目经历

### ComfyUI 自动化图像管线

#### 真实开发过程

- **起点**：基于社区已有的 D站标签插件进行改造，非从零开发
- **协作方式**：全程 AI 辅助开发（AI Coding / Vibecoding），与 Hermes 对话中完成
- **周期**：仅几天时间
- **性质**：在开源生态中快速理解、改造、扩展——而非闭门造轮子

#### 核心能力体现

不在于代码量，而在于：
- 对 ComfyUI 插件架构和 D站 API 的快速理解
- 知道改什么、怎么改——JS 前端 Widget 通信、Python 后端节点逻辑
- 能用 AI 高效完成"理解→修改→测试"循环

#### 简历策略

- 诚实表述为"基于开源项目改造 + AI 辅助开发"
- 强调交付速度和对生态的理解深度
- 不可夸大代码量或独立开发程度
